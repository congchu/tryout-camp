# 포트폴리오 구조 스펙

> 좋은 포트폴리오에 들어가야 할 항목 정의

---

## 철학: 좋은 포트폴리오란?

### 3가지 레이어

```
1. 의도 (브랜딩) → WHY: 나는 어떤 사람으로 보여지고 싶은가
2. 구조 → WHAT: 그 의도를 전달하려면 뭘 보여줘야 하는가
3. UI/UX → HOW: 그걸 어떻게 표현할 것인가
```

**순서가 중요하다.** 의도 → 구조 → UI/UX 순으로 설계해야 함.
거꾸로 하면 속 빈 포트폴리오가 됨.

### 포트폴리오는 "설득하는 글"이다

- 주장이 있어야 함 (나는 이런 가치를 줄 수 있다)
- 설득이 있어야 함 (그 근거가 여기 있다)

### 취업 vs 프리랜서

| 타겟 | 설득 방식 | 이유 |
|------|----------|------|
| 취업 | 직접 말하기 | 가능성을 봄, 확신 필요 |
| 프리랜서 | 간접 증명 | 퀄리티가 핵심, 결과물로 보여줌 |

**이 문서는 프리랜서용 기준.**
→ 결과물 퀄리티로 증명하는 방식

### 그래도 브랜딩은 중요하다

간접 증명이 중요하지만, 브랜딩(의도)이 기반이 되어야 함.
- 브랜딩 없이 결과물만 나열 → 뭘 잘하는 사람인지 모호
- 브랜딩이 있으면 → 결과물이 그 브랜딩을 뒷받침

## 1. 기본 정보

| 항목 | 필수 | 설명 |
|------|------|------|
| 이름 | ✅ | 포트폴리오 주인이 누군지 |
| 직함 | ✅ | 뭘 하는 사람인지 3초 안에 파악 |
| 태그라인 | ✅ | 직함만으론 차별화 안 됨, 개성 표현 |
| 상세 소개 | ✅ | About 섹션용, 신뢰감 형성 (2-3문장) |
| 프로필 이미지 | ✅ | 얼굴 or 아바타 |

---

## 2. 스킬

| 항목 | 필수 | 설명 |
|------|------|------|
| 스킬 태그 | ✅ | 어떤 툴/기술을 다루는지 (Figma, Photoshop 등) |
| 카테고리 분류 | 선택 | 디자인 툴 / 그래픽 / 협업 등 |

---

## 3. 프로젝트

### 개수
- 최소: 3개 (너무 적으면 경험 없어 보임)
- 최대: 6개 (너무 많으면 집중 안 됨)
- 권장: 4개

### 각 프로젝트 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 프로젝트명 | ✅ | 뭘 만들었는지 한눈에 |
| 카테고리 | ✅ | 어떤 분야인지 (UI/UX, 브랜딩, 웹 등) |
| 한 줄 설명 | ✅ | 뭘 했는지 간단히 |
| 썸네일 이미지 | ✅ | 시각적으로 보여줘야 포트폴리오 |
| 외부 링크 | 선택 | 실제 작업물 확인 가능하면 신뢰↑ |

---

## 4. 연락처

| 항목 | 필수 | 설명 |
|------|------|------|
| 이메일 | ✅ | 연락받을 수단 필수, 없으면 포트폴리오 의미 없음 |
| 소셜 링크 | ✅ | 요즘은 소셜이 신뢰 지표 (LinkedIn, Instagram 등) |
