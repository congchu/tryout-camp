# 결정사항

이 문서는 중요한 결정들과 그 이유를 기록한다. "왜 이렇게 했지?" 할 때 보는 문서.

---

## 1. 아키텍처 결정

### 캠프 프로젝트 분리

**상황**: 메인 사이트에 통합 vs 별도 프로젝트

**결정**: 별도 프로젝트로 분리

**이유**:
- 메인 사이트 비대화 방지
- 빠르게 실험해야 하는데 메인에 영향 주기 싫음
- 독립적으로 배포/관리 가능

---

### Firebase 선택

**상황**: 인증 + DB를 뭘로 할지

**결정**: Firebase (Auth + Firestore)

**이유**:
- 빠른 셋업 (몇 분)
- 무료 티어로 MVP 가능
- 실시간 기능 내장
- Next.js 통합 쉬움

**대안**: Supabase, 자체 DB → 셋업 복잡

---

### Slack 알림

**상황**: 신청/미션 제출 알림 방법

**결정**: Slack Webhook

**이유**:
- 어드민 페이지 안 만들어도 됨
- 실시간 확인
- 설정 간단

---

## 2. 제품 결정

### 워크북 형태

**결정**: 웹 기반 텍스트 워크북

**이유**:
- 영상보다 빠르게 제작
- 수정 쉬움
- 참가자가 자기 속도로 진행

**대안**: 영상 → 제작 오래 걸림

---

### 챗봇 신청

**결정**: 챗봇 형태로 신청 받기

**이유**:
- 폼보다 친근함
- 실험마켓 브랜드와 맞음
- 대화하며 정보 수집

---

### 6일 구성

**결정**: 캠프 기간 6일

**이유**:
- 5일은 빡빡, 7일은 주말 부담
- 하루 30분 내외 가능
- 일주일 안에 끝남

---

### Day별 순차 공개

**결정**: Day별로 잠금, 순차 공개

**이유**:
- 매일 진행하는 느낌
- 앞서가는/뒤처지는 격차 줄임
- "함께 하는" 챌린지 느낌

---

## 3. 아직 결정 안 한 것

- 유료 캠프 가격
- 알림 자동화 방식
- 메인 플랫폼 연동
