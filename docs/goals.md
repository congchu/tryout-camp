# 목표

이 문서는 캠프 프로젝트의 목표와 방향을 정의한다. "우리가 뭘 달성하려고 하는지", "지금 어디까지 왔는지" 여기서 확인할 수 있다.

---

## 1. 왜 목표 문서가 필요해?

프로젝트를 하다 보면 방향을 잃기 쉽다:
- "이거 왜 하고 있었지?"
- "지금 뭐가 중요한 거지?"
- "다음에 뭘 해야 하지?"

목표가 명확하면:
- 우선순위를 정할 수 있다
- 뭘 해야 하고 뭘 안 해도 되는지 알 수 있다
- 진행 상황을 측정할 수 있다

---

## 2. 캠프 프로젝트의 목표

### 큰 그림

**"실험마켓의 MVP이자 유입 채널로서 캠프를 성공적으로 운영한다"**

풀어서 말하면:
- 새로운 주제를 **빠르게 테스트**할 수 있는 구조를 만든다
- 콜라보를 통해 **새로운 고객**을 만난다
- 잘 된 캠프는 **정규 코스로 확장**한다

### 성공의 정의

캠프가 성공했다는 건 뭘까?

| 관점 | 성공 기준 |
|------|-----------|
| 참가자 | "해볼 만했다", "결과물이 나왔다" |
| 운영자 | "효율적으로 운영했다", "재사용 가능한 구조가 생겼다" |
| 비즈니스 | "유입이 생겼다", "다음 단계(정규 코스)로 연결됐다" |

---

## 3. 핵심 지표 (KPI)

뭘 측정해야 캠프가 잘 되고 있는지 알 수 있을까?

### 참여 지표

| 지표 | 설명 | 목표 |
|------|------|------|
| 신청자 수 | 랜딩 → 신청 완료 | 캠프마다 다름 |
| 완주율 | 마지막 Day 미션 제출 / 신청자 | 50% 이상 |
| 일별 이탈률 | 각 Day에서 이탈하는 비율 | Day3 이후 안정화 |

### 품질 지표

| 지표 | 설명 | 목표 |
|------|------|------|
| 결과물 완성도 | 실제로 쓸 수 있는 결과물인지 | 70% 이상 |
| 만족도 | 마지막 설문 점수 | 4/5 이상 |
| 추천 의향 | NPS | +30 이상 |

### 비즈니스 지표

| 지표 | 설명 | 목표 |
|------|------|------|
| 전환율 | 캠프 → 정규 코스/커뮤니티 가입 | 10% 이상 |
| CAC | 참가자 1명당 획득 비용 | - |
| ROI | 투자 대비 수익 | - |

## 4. 현재 상태

### Phase 1: MVP 검증 (현재 여기)

**목표**: "캠프라는 형태가 작동하는지 확인"

- [x] 첫 번째 캠프 기획 (AI 포트폴리오)
- [x] 콜라보 파트너 확보 (크크웍스)
- [x] 워크북 시스템 구축
- [x] 6일 캠프 운영 완료
- [ ] 데이터 분석 및 회고
- [ ] 개선점 도출

**검증하려는 것**:
- 5-7일 단기 챌린지가 참여율이 높은지
- 워크북 형태가 효과적인지
- 콜라보가 유입에 도움이 되는지

### Phase 2: 반복 및 개선 (다음)

**목표**: "검증된 구조로 2-3개 캠프 더 운영"

- [ ] 1기 회고 기반 개선
- [ ] 2기 또는 새 주제 캠프 기획
- [ ] 캠프 제작 프로세스 정립
- [ ] 템플릿화 (랜딩, 워크북 등)

### Phase 3: 시스템화 (나중에)

**목표**: "캠프를 빠르게 찍어낼 수 있는 구조"

- [ ] 캠프 제작 자동화
- [ ] 알림/리마인더 자동화
- [ ] 메인 플랫폼 연동
- [ ] 데이터 대시보드

## 7. 안 하는 것

목표만큼 중요한 게 "안 하는 것"을 명확히 하는 거다.

**지금은 안 하는 것:**
- 유료 캠프 (MVP 검증 단계에서는 무료/저가로)
- 복잡한 기능 (알림 자동화, 대시보드 등)
- 여러 캠프 동시 운영 (한 번에 하나씩)
- 메인 플랫폼 연동 (아직 이름)
