# 캠프 현황

이 문서는 모든 캠프의 현황을 추적한다. 어떤 캠프가 있고, 각각 어디까지 진행됐는지 여기서 확인할 수 있다.

---

## 1. 캠프 상태 정의

캠프는 이런 단계를 거친다:

```
💡 기획 → 🛠 제작 → 📢 모집 → 🚀 운영 → ✅ 완료 → 📦 아카이브
```

| 상태 | 의미 | 이 상태에서 하는 일 |
|------|------|---------------------|
| 💡 기획 | 아이디어 단계 | 주제 선정, 콜라보 파트너 협의, 일정 확정 |
| 🛠 제작 | 만드는 중 | 레슨 플랜 작성, Day별 콘텐츠, 랜딩 페이지 |
| 📢 모집 | 참가자 모집 | 홍보, 광고, 신청 받기 |
| 🚀 운영 | 진행 중 | Day별 오픈, 미션 확인, Q&A 대응 |
| ✅ 완료 | 운영 끝남 | 데이터 정리, 회고 작성, 개선점 도출 |
| 📦 아카이브 | 정리 완료 | 콘텐츠 재사용 검토, 다음에 반영 |

### ✅ 완료된 캠프

| 항목 | 내용 |
|------|------|
| 상태 | ✅ 완료 |
| 콜라보 | 크크웍스 × 실험마켓 |
| 기간 | 6일 |
| URL | `/cc/ai-portfolio` |

**왜 이 주제?**

크크웍스(Kookworks)는 개발/디자인 교육을 하는 브랜드다. "비전공자도 포트폴리오를 만들 수 있다"는 메시지가 실험마켓의 "작게 실험"과 맞았다.

AI 도구(Lovable, v0, Bolt 등)가 발전하면서 코딩 없이도 웹사이트를 만들 수 있게 됐다. 이걸 활용해서 "진짜 비전공자가 6일 만에 포트폴리오를 만들 수 있는지" 실험해보자는 게 시작이었다.

| 3 | 콘텐츠 완성하기 | 내 정보 + 이미지 채우기 |
| 4 | 배포하기 | 실제 URL로 공개 |

## 3. 캠프 라이프사이클 상세

각 단계에서 뭘 해야 하는지 체크리스트.

### 💡 기획 단계

새 캠프를 시작하기 전에 확정해야 할 것들:

- [ ] **주제**: 뭘 만드는 캠프인지
- [ ] **타겟**: 누구를 위한 건지
- [ ] **기간**: 며칠짜리인지 (보통 5-7일)
- [ ] **콜라보 파트너**: 누구와 함께 하는지 (단독도 가능)
- [ ] **일정**: 언제 시작하는지
- [ ] **목표**: 이 캠프로 뭘 검증하려는지

### 🛠 제작 단계

캠프 콘텐츠를 만드는 단계:

- [ ] **레슨 플랜**: 각 Day별 목표, 핵심 메시지, 스텝 구성
- [ ] **워크북 콘텐츠**: Day별 실제 내용 작성
- [ ] **미션 설계**: 매일 제출할 과제
- [ ] **랜딩 페이지**: 캠프 소개 + 신청 페이지
- [ ] **테스트**: 전체 플로우 확인

### 📢 모집 단계

참가자를 모으는 단계:

- [ ] **홍보 채널**: 어디서 홍보할지 (SNS, 커뮤니티, 광고 등)
- [ ] **홍보 소재**: 이미지, 카피 등
- [ ] **신청 폼 오픈**: 신청 받기 시작
- [ ] **참가자 안내**: 신청자에게 시작 안내

### 🚀 운영 단계

캠프가 진행되는 동안:

- [ ] **Day별 오픈**: 매일 새 콘텐츠 공개
- [ ] **미션 확인**: 제출된 미션 확인 (Slack 알림)
- [ ] **Q&A 대응**: 참가자 질문 답변
- [ ] **리마인더**: 미제출자에게 알림 (선택)

### ✅ 완료 단계

캠프가 끝난 후:

- [ ] **완주자 축하**: 완주 메시지, 인증서 등
- [ ] **데이터 정리**: 신청자, 완주율, 피드백 등
- [ ] **회고 작성**: 뭐가 잘 됐고, 뭘 개선할지
- [ ] **개선점 도출**: 다음 캠프에 반영할 것

### 📦 아카이브 단계

완전히 정리가 끝난 후:

- [ ] **콘텐츠 재사용 검토**: 다른 캠프에 쓸 수 있는지
- [ ] **문서 정리**: 이 문서에 결과 기록
- [ ] **다음 캠프 기획에 반영**

## 4. 새 캠프 추가하는 법

1. 이 문서의 "기획 중인 캠프" 섹션에 아이디어 추가
2. 기획이 확정되면 "캠프 목록"에 새 섹션 추가
3. 상태를 💡 → 🛠 → 📢 → 🚀 → ✅ → 📦 순서로 업데이트
4. 각 단계의 체크리스트 활용
