# 트러블슈팅: Day 3 콘텐츠 작성 실패

## 발생한 문제

`/write day3` 실행 시 **스펙과 완전히 다른 방향**으로 콘텐츠 작성됨

### 내가 작성한 것 (잘못됨)
```
1. Day 3 시작하기
2. 프로젝트 이미지 준비하기  ← 스펙에 없음
3. 텍스트 콘텐츠 채우기      ← 스펙에 없음
4. 이미지 교체하기          ← 스펙에 없음
5. 최종 점검
```

### 스펙에 정의된 것 (정답)
```
1. 목적 정하기 (포트폴리오론)
2. 나 위키 만들기
3. 인터뷰로 카피 뽑기
4. 웹사이트에 적용
```

→ **완전히 다른 콘텐츠**가 만들어짐

---

## 원인 분석

### 직접 원인
**스펙 문서를 확인하지 않음**

- `_content/specs/day-03.md` 파일이 존재했음
- CLAUDE.md에 "콘텐츠 작성 전 스펙 확인 필수" 규칙 있었음
- 그런데 나(Claude)는 이걸 무시하고 로드맵만 보고 추측으로 작성함

### 근본 원인
**에이전트 가이드에 스펙 확인 절차가 없음**

`.claude/agents/writer.md`의 참조 파일 섹션:
```markdown
## 참조 파일
1. 로드맵: CLAUDE.md → 5일 챌린지 로드맵 섹션
2. 리서치: sample/research-day{n}.md
3. 이전 Day: day{n-1}.ts
4. 타입 정의: types.ts
```

→ **스펙 문서 참조가 빠져있음!**

`.claude/skills/write.md`의 실행 절차:
```markdown
## 실행 절차
1. .claude/agents/writer.md 가이드 읽기
2. CLAUDE.md에서 해당 Day 주제 확인
3. sample/research-day{n}.md 리서치 자료 읽기
4. 이전 Day 콘텐츠 확인
5. 콘텐츠 작성
6. day{n}.ts 저장
```

→ **스펙 확인 단계가 없음!**

## 문제 발생 흐름

```
사용자: "/write day3 해봐"
    ↓
Claude: writer.md 가이드 읽음
    ↓
Claude: CLAUDE.md 로드맵 확인 → "콘텐츠 완성하기"
    ↓
Claude: Day 2 마지막 확인 → "내 정보와 이미지 채워서"
    ↓
Claude: (스펙 확인 안 함) ← 여기가 문제!
    ↓
Claude: 추측으로 "이미지 준비 → 텍스트 채우기" 구조 작성
    ↓
잘못된 콘텐츠 생성
    ↓
사용자: "WORKLOG 보니까 다른 방향으로 논의했는데?"
    ↓
Claude: 스펙 문서 발견 → 다시 작성
```

## 해결책

### 1. writer.md 참조 파일에 스펙 추가

```markdown
## 참조 파일

1. **⚠️ 스펙 (필수)**: `_content/specs/day-{n}.md`  ← 추가!
2. 로드맵: CLAUDE.md
3. 리서치: sample/research-day{n}.md
4. 이전 Day: day{n-1}.ts
5. 타입 정의: types.ts
```

### 2. write.md 실행 절차에 스펙 확인 추가

```markdown
## 실행 절차

1. .claude/agents/writer.md 가이드 읽기
2. **⚠️ 스펙 확인: `_content/specs/day-{n}.md` 읽기**  ← 추가!
3. 스펙이 없으면 사용자에게 방향 확인
4. CLAUDE.md에서 해당 Day 주제 확인
5. ...
```

### 3. 체크리스트에 스펙 확인 항목 추가

```markdown
## 체크리스트

- [ ] **⚠️ 스펙 문서 확인** (specs/day-{n}.md)  ← 맨 위에!
- [ ] 스펙의 핵심 메시지 파악
- [ ] 스펙의 Step 구조 따르기
- [ ] ...
```

## 교훈

1. **문서가 있어도 참조하지 않으면 소용없음**
   - CLAUDE.md에 규칙 추가 ≠ 실제로 따름
   - 에이전트 가이드에도 명시해야 함

2. **에이전트 가이드는 "실행 매뉴얼"이어야 함**
   - 빠진 단계가 있으면 그 단계는 실행 안 됨
   - 모든 필수 단계를 명시적으로 포함해야 함

3. **스펙 > 로드맵 > 추측**
   - 스펙이 있으면 스펙 따르기
   - 스펙이 없으면 로드맵 참고
   - 둘 다 없으면 사용자에게 확인
