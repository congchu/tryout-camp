# 증거와 설득 (토론 기록)

## 발단

**나:**
이게 그리고 추상화 레벨에 따라 강도가 좀 다를거 같아. 예를 들어, 디자이너는 결과물을 정말로 그대로 눈으로 보여줄수 있잖아. 거기에 물론 설계의도 같은것들이 들어가야되긴 하겠지만, 여튼 그래서 직접 증거 < 간접 증거가 더 의미가 있을거 같아. 굳이 내가 막 저 이런 사람이에요!!! 이렇게 1차원적으로 설명할 필요는 없으니까.

반면에 개발자는 결과를 보여주는게 꽤 어럽다고 생각해. 클라이언트들이 코드를 보진 못하거든. 웹사이트 결과물을 보긴할수 있지만 그게 순수하게 개발자의 실력일까? 그래서 주장을 설득력있게 글로 풀어내는 부분이 더 중요하다고 생각해. 네 생각은 어때?

---

**Claude:**
동의해. 정리하면:

| 직군 | 결과물 가시성 | 증명 방식 |
|------|--------------|----------|
| 디자이너 | 👁️ 눈에 바로 보임 | 간접 증명 (보여주면 끝) |
| 개발자 | 🔒 코드는 안 보임 | 직접 설명 필요 |
| 마케터 | 📊 수치로 보임 | 중간? |

### 디자이너
```
결과물 = 증거
"이거 제가 만들었어요" → 끝
```

### 개발자
```
결과물 ≠ 온전한 증거
- 웹사이트 보여줘도 → 디자이너 덕? 기획자 덕? 개발자 덕?
- 코드 보여줘도 → 클라이언트가 못 읽음
```

그래서 개발자는 **"왜 이게 어려웠고, 내가 뭘 해결했는지"** 글로 풀어야 함.

개발자도 간접 증명 가능하긴 해:
- GitHub 기여도
- 오픈소스 프로젝트
- 기술 블로그

**근데 이건 "개발자 커뮤니티"에서만 통함.**

클라이언트는 GitHub 못 읽어. 그래서 클라이언트 대상 프리랜서 개발자는 **"번역 능력"**이 필요함.

```
기술적 성과 → 클라이언트가 이해하는 언어로 번역
```

---

**Claude:**
그럼 마케터는? 수치로 증명 가능하니까 디자이너에 가깝지 않을까?

```
"이 캠페인으로 전환율 30% 올렸어요" → 숫자가 증거
```

---

**나:**
응 맞아. 수치가 있으면 너무 쉽지. 근데 없는 마케터들도 많아서... 취준생일수도 있고. 그러면 어떻게 해야하나?

---

**Claude:**
수치가 없으면 "결과물"이 없는 거잖아. 그럼 뭘로 증명해?

### 가능한 방법들

| 방법 | 설명 |
|------|------|
| 가상 캠페인 기획서 | 실행 안 했지만 전략/논리 보여줌 |
| 개인 SNS 운영 | 팔로워, 조회수 (작은 수치라도) |
| 콘텐츠 퀄리티 | 글 잘 쓰는지, 카피라이팅 실력 |
| 케이스 스터디 분석 | 다른 회사 캠페인 분석 → 사고방식 보여줌 |

결국 이렇게 되는 것 같아:

```
수치 있는 마케터 → 디자이너처럼 (간접 증명)
수치 없는 마케터 → 개발자처럼 (직접 설명, 논리 전개)
```

---

**나:**
결국은 증명할만한 증거가 있으면 그걸 최대한 활용하되, 없으면 어떻게든 설득해야한다라고 볼 수 있을까? 프로젝트가 여러개면 쉽지만, 1개라면 그걸로 어떻게든 설득해야하니까.

---

**Claude:**
정확해. 직군 얘기는 사실 표면이고, 근본 원칙은:

```
증거 있으면 → 보여준다 (간접 증명)
증거 없으면 → 설득한다 (직접 설명)
```

## 결론

### 근본 원칙

```
증거 있으면 → 보여준다 (간접 증명)
증거 없으면 → 설득한다 (직접 설명)
```

### 직군별 가시성 (표면)

| 직군 | 가시성 | 기본 증명 방식 |
|------|--------|---------------|
| 디자이너 | 높음 | 결과물 |
| 마케터 (수치 있음) | 높음 | 수치 |
| 마케터 (수치 없음) | 낮음 | 논리/기획력 |
| 개발자 | 낮음 | 논리/설명 |

### 변수들

| 변수 | 많음/높음 | 적음/낮음 |
|------|----------|----------|
| 결과물 가시성 | 보여주면 됨 | 설명해야 함 |
| 프로젝트 개수 | 골라서 보여줌 | 하나로 깊게 파야 함 |
| 수치/성과 | 숫자가 증명 | 논리로 설득 |

### 포트폴리오 전략

```
1. 내 증거 카드 점검 (뭐가 있나?)
2. 있으면 → 최대한 활용 (잘 보여주기)
3. 없으면 → 논리로 채우기 (왜 이렇게 했는지, 뭘 배웠는지)
```
