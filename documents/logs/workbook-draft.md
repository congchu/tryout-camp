## 챌린지 소개

### 왜 포트폴리오인가?

프리랜서에게 포트폴리오는 **명함이자 영업사원**입니다.
클라이언트는 "이 사람이 어떤 사람인지"보다 **"그래서 퀄리티를 낼 수 있어?"**를 봅니다.

결과물로 증명하는 것. 그게 프리랜서 포트폴리오의 핵심입니다.

### 이 챌린지에서 만드는 것

- **나만의 웹 포트폴리오** (코딩 없이 AI로!)
- 평생 무료 호스팅
- 나만의 도메인 연결 가능 (선택)

### 대상

- 포트폴리오 필요한데 계속 미뤄온 프리랜서
- 코딩/디자인 못해도 OK
- 하루 30분 투자 가능한 분

## Day 1: 브랜딩 (WHY)

### 오늘의 질문
> "나는 어떤 사람으로 보여지고 싶은가?"

### 왜 브랜딩이 먼저인가?

```
❌ 예쁜 템플릿 찾아서 → 내 내용 넣기
✅ 내 브랜딩 정하고 → 그에 맞는 구조와 디자인 선택
```

순서가 중요합니다. 거꾸로 하면 속 빈 포트폴리오가 됩니다.

### 미션

**1. 자기소개 3문장 작성**

아래 질문에 답해보세요:
- 나는 무슨 일을 하는 사람인가?
- 어떤 문제를 해결해주는가?
- 왜 나를 선택해야 하는가?

예시:
> "저는 브랜드의 첫인상을 만드는 디자이너입니다.
> 복잡한 정보를 심플하게 정리하는 걸 잘합니다.
> 사용자가 '쉽다'고 느끼게 만드는 게 제 목표예요."

**2. 나를 표현하는 키워드 5개**

예시: #미니멀 #사용자중심 #깔끔한정리 #B2B전문 #빠른커뮤니케이션

**3. 직함 + 태그라인**

형식: `[직함] | [태그라인]`

예시:
- UI/UX 디자이너 | 복잡함을 단순하게
- 풀스택 개발자 | 빠르게 만들고, 제대로 굴리는
- 콘텐츠 마케터 | 읽히는 글을 씁니다

### 제출

슬랙 #챌린지-미션 채널에 아래 형식으로 제출:

```
[Day 1 제출]

1. 자기소개 3문장
   (여기에 작성)

2. 나를 표현하는 키워드 5개
   #키워드1 #키워드2 ...

3. 직함 + 태그라인
   (여기에 작성)
```

### 팁

- 완벽하지 않아도 됩니다. 나중에 계속 다듬어요.
- AI 도움 받아도 OK! (ChatGPT에게 "내 강점을 정리해줘" 요청)
- 다른 분들 제출물 보면서 영감 얻어도 좋아요

## Day 2: 구조 설계 (WHAT)

### 오늘의 질문
> "나의 브랜딩을 전달하려면 뭘 보여줘야 하나?"

### 포트폴리오 필수 구성요소

| 항목 | 설명 |
|------|------|
| 이름 + 직함 | 누구인지 3초 안에 파악 |
| 태그라인 | 차별화 포인트 |
| 소개 | About 섹션, 2-3문장 |
| 프로필 이미지 | 얼굴 or 아바타 |
| 스킬 태그 | 사용 가능한 툴/기술 |
| 프로젝트 | **3~6개** (너무 적으면 경험 없어 보임, 너무 많으면 집중 안 됨) |
| 연락처 | 이메일 필수, 소셜 링크 |

### 미션

**1. 프로젝트 리스트 작성**

최소 3개, 최대 6개 프로젝트 선정

각 프로젝트마다:
- 프로젝트명
- 카테고리 (예: UI/UX, 브랜딩, 웹개발 등)
- 한 줄 설명
- 썸네일 이미지 (있으면)
- 외부 링크 (있으면)

**2. 스킬 태그 정리**

내가 다룰 수 있는 툴/기술 나열

예시:
- 디자이너: Figma, Photoshop, Illustrator, After Effects
- 개발자: React, Node.js, Python, AWS
- 마케터: GA4, Meta Ads, Notion, Canva

**3. 연락처 정리**

- 이메일 (필수)
- 소셜 링크 (LinkedIn, Instagram, Twitter 등)

### 제출

```
[Day 2 제출]

1. 프로젝트 리스트 (3~6개)
   - 프로젝트1: [제목] / [카테고리] / [한줄설명]
   - 프로젝트2: ...
   - ...

2. 스킬 태그
   (쉼표로 나열)

3. 연락처
   - 이메일:
   - 소셜:
```

### 팁

- 프로젝트 이미지가 없다면? 목업 만들어도 OK
- AI로 목업 이미지 생성 가능 (내일 알려드림)

### 오늘의 목표
> 어제 만든 결과물 다듬기

### 미션

**1. 문제점 파악**

어제 만든 포트폴리오에서 수정하고 싶은 부분 3가지 이상 찾기

**2. AI에게 수정 요청**

예시 프롬프트:
- "색상을 더 따뜻하게 바꿔줘"
- "프로젝트 카드 간격을 더 넓게"
- "폰트를 더 현대적으로"
- "프로필 이미지를 동그랗게"

**3. 최소 3번 수정 반복**

AI → 결과 확인 → 수정 요청 → 결과 확인 → ...


## Day 5: 배포 및 완성

### 오늘의 목표
> 세상에 공개하기!

### 배포 옵션

**1. Lovable 기본 배포 (가장 쉬움)**
- Lovable에서 바로 배포 가능
- URL: `yourname.lovable.app` 형태

**2. Vercel 배포 (추천)**
- 무료, 빠름, 안정적
- URL: `yourname.vercel.app` 형태
- 나중에 커스텀 도메인 연결 가능

**3. 나만의 도메인 연결 (보너스)**
- 1년에 1~2만원 정도
- `yourname.com` 형태로 더 프로페셔널

### 미션

**1. 배포 완료**

선택한 방법으로 배포하고 URL 확보

**2. 최종 점검**

체크리스트:
- [ ] 모바일에서도 잘 보이나?
- [ ] 링크들 다 작동하나?
- [ ] 오타 없나?
- [ ] 이미지 다 로딩되나?

**3. SNS/카톡에 공유 (선택)**

만든 포트폴리오 자랑해보세요!

## 부록: AI 프롬프트 팁

### 잘 되는 프롬프트
- 구체적인 수치 (색상 코드, 크기 등)
- 레퍼런스 사이트 링크
- "~하지 마" 금지사항 명시

### 안 되는 프롬프트
- "예쁘게 해줘" (주관적)
- "아름답게" (AI마다 다름)
- 한 번에 너무 많은 요청

### 수정 요청 예시
```
- "배경색을 #F5F5F5로 바꿔줘"
- "프로젝트 섹션을 2열 그리드로"
- "About 섹션은 그대로 두고 색상만 변경"
- "버튼 모서리를 둥글지 않게 (border-radius 없이)"
```
