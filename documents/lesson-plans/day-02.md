# Day 2: 화려한 포폴 따라만들기

## 핵심 메시지
Day 1 방식의 한계를 넘어서, **멋진 사이트를 완벽하게 복사**하는 방법을 배운다.
스크린샷이 아닌 **설계도(소스코드)**를 가져오면 "똑같이" 가능하다.

---

## 학습 목표
1. Day 1 방식의 한계 이해 (비슷하게 vs 똑같이)
2. GitHub에서 오픈소스 코드 가져오는 개념 이해
3. Codex로 레퍼런스 복사 + 내 정보로 수정

---

## Day 1 vs Day 2

| 구분 | Day 1 | Day 2 |
|------|-------|-------|
| 참고 방식 | 스크린샷 + 설명 | **설계도(GitHub) 직접 가져옴** |
| 결과물 | "비슷하게" | **"똑같이" 복사** |
| 도구 | AI 빌더 (웹) | Codex (내 컴퓨터) |
| 한계 | 애니메이션 어색 | 코드 그대로라 완벽 |

---

## 흐름

### Step 1. Day 1 방식의 한계
- AI 빌더 결과물은 비슷비슷함
- 스크린샷으로는 "비슷하게"는 되지만 "똑같이"는 어려움
- 특히 움직임(스크롤, 호버, 전환)이 어색

### Step 2. 멋진 사이트 똑같이 만들기
- GitHub = 개발자들이 코드를 공유하는 "도서관"
- 공개된 설계도를 마음대로 가져올 수 있음
- Codex가 가져와서 실행/수정 도와줌

### Step 3. Codex 설치하기
- OpenAI 코딩 전용 앱 (맥/윈도우)
- ChatGPT 계정으로 로그인
- 프로젝트 폴더 선택

### Step 4. 레퍼런스 복사하기
- 템플릿 모음에서 선택
- "이거 복사해서 실행해줘 [GitHub URL]"
- 2~5분 대기 → localhost 링크 클릭

### Step 5. 내 정보로 바꾸기
- 텍스트 정보 입력
- 이미지 드래그 앤 드롭
- 기존 자료(PDF, PPT 등) 업로드 가능

### Step 6. 한글 폰트 적용
- 눈누에서 무료 폰트 찾기
- 폰트 URL 복사해서 Codex에 요청
