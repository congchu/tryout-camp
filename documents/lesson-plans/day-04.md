# Day 4: 링크로 공유하기

## 핵심 메시지
Day 1-3에서 만든 포트폴리오는 **내 컴퓨터에서만** 보인다.
오늘은 **링크 하나로 누구에게든 공유**할 수 있게 만든다.

> 카톡에 링크 붙여넣으면, 미리보기까지 예쁘게 뜨는 상태!

---

## 3단계 흐름

### Step 1. 공유 링크 만들기
- Codex에 "Netlify 배포하고 싶어" 요청
- Codex가 프로젝트 파악 → 알아서 빌드 → 가이드
- 결과: **진짜 URL** 생성

### Step 2. 공유 미리보기 설정
- 카톡/슬랙에서 링크 공유할 때 뜨는 정보
- 제목, 설명, 이미지 (OG 태그)
- Codex에 요청하면 알아서 설정

### Step 3. 공유해보기
- 링크 복사 → 카톡/슬랙에 붙여넣기
- 미리보기 확인
- 완성!

---

## 예상 어려움 + 대응 방법

### Step 1. 공유 링크 만들기
| 어려움 | 대응 |
|--------|------|
| **Netlify가 뭔지 모름** | "무료로 링크 만들어주는 서비스"라고만 설명 |
| **빌드 에러** | 에러 메시지 Codex에 붙여넣기 → 해결 |
| **어떤 폴더 올려야 하는지 모름** | Codex가 알려줌. 따라하면 됨 |
| **Netlify 영어라 어려움** | 스크린샷으로 가이드 제공 |

### Step 2. 공유 미리보기 설정
| 어려움 | 대응 |
|--------|------|
| **OG 태그가 뭔지 모름** | 설명 안 함. "공유 미리보기"라고만 |
| **이미지 어떤 거 써야 하지** | 프로필 사진 또는 대표 작업물 추천 |
| **설정했는데 안 뜸** | 캐시 문제. 카카오톡 캐시 초기화 방법 안내 |

### Step 3. 공유해보기
| 어려움 | 대응 |
|--------|------|
| **미리보기가 안 뜸** | 캐시 문제 or 설정 오류 → Codex에 물어보기 |
| **이미지가 깨짐** | 이미지 경로/크기 문제 → Codex가 수정 |

## FAQ (예상 질문)

### 링크 관련
**Q. 링크 주소가 이상해요 (netlify.app)**
→ 무료 버전은 이렇게 나옴. 나중에 커스텀 도메인 연결 가능.

**Q. 링크 주소 바꿀 수 있어요?**
→ Netlify 설정에서 사이트 이름 변경 가능. (예: hongildong.netlify.app)

**Q. 수정하면 자동으로 반영돼요?**
→ 아니요. 수정 후 다시 배포해야 함. (다시 "배포해줘" 요청)

### 미리보기 관련
**Q. 카톡에 미리보기가 안 떠요 / 예전 정보가 떠요**
→ 캐시 문제. 아래 "캐시란?" 참고.

**Q. 이미지 비율이 이상해요**
→ OG 이미지 권장 크기: 1200x630px. Canva에서 만들거나 Codex에 요청.

## 캐시(Cache)란?

### 쉽게 설명
> **캐시 = 저장해둔 복사본**

카카오톡이 링크를 처음 볼 때, 미리보기 정보를 **저장해둡니다.**
그래서 내가 수정해도 **예전 정보가 계속 뜨는 것**처럼 보여요.

### 해결 방법: 카카오톡 캐시 초기화

1. [카카오 캐시 초기화 도구](https://developers.kakao.com/tool/debugger/sharing) 접속
2. 내 포트폴리오 URL 입력
3. **"캐시 초기화"** 버튼 클릭
4. 카톡에 다시 링크 붙여넣기 → 새 정보로 뜸!

### 왜 이런 게 있을까?
- 매번 새로 불러오면 느려지니까
- 한 번 저장해두고 재사용하는 것
- 대부분의 웹서비스가 이렇게 동작함

> 💡 미리보기 수정했는데 안 바뀌면? **무조건 캐시 초기화!**
